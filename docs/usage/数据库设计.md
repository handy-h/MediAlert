## 2. 表结构

### 2.1 medications（药品表）

存储所有药品的基本信息、库存和用药频率。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 自增主键 |
| genericName | TEXT | NOT NULL | 药品通用名 |
| brandName | TEXT | NULL | 商品名 |
| specification | TEXT | NULL | 规格文本（仅显示） |
| packageUnit | TEXT | NOT NULL | 包装单位：盒/支/瓶 |
| dosageForm | TEXT | NOT NULL | 剂型：片/粒/ml/g |
| packageSize | INTEGER | NOT NULL | 每包装含量 |
| currentStock | REAL | NOT NULL | 当前库存（总剂型数） |
| frequencyType | TEXT | NOT NULL | 频率类型：EVERY_X_DAYS / EVERY_XTH_DAY |
| frequencyValue | INTEGER | NOT NULL | 频率值 |
| dailyDosage | REAL | NOT NULL | 每次剂量 |
| startDate | TEXT | NULL | 开始日期（ISO 格式：yyyy-MM-dd） |
| isActive | INTEGER | NOT NULL DEFAULT 1 | 是否启用：1=启用，0=停用 |
| createdAt | INTEGER | NOT NULL DEFAULT 0 | 创建时间戳（毫秒） |

#### 索引
```sql
-- 按状态查询索引
CREATE INDEX index_medications_isActive ON medications(isActive);

-- 按创建时间排序索引
CREATE INDEX index_medications_createdAt ON medications(createdAt);
```

#### 示例数据
```sql
INSERT INTO medications (
    genericName, brandName, specification, 
    packageUnit, dosageForm, packageSize,
    currentStock, frequencyType, frequencyValue, dailyDosage,
    startDate, isActive, createdAt
) VALUES (
    '厄贝沙坦', '安博维', '80/12.5',
    '盒', '片', 14,
    34.0, 'EVERY_X_DAYS', 1, 1.0,
    '2026-06-01', 1, 1717776000000
);
-- 表示：厄贝沙坦，每盒14片，当前库存34片（2盒+6片）
-- 每天1片，从2026-06-01开始
```

### 2.2 stock_logs（库存日志表）

记录所有库存变更操作，用于审计和追溯。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 自增主键 |
| medicationId | INTEGER | NOT NULL FOREIGN KEY | 关联药品ID |
| type | TEXT | NOT NULL | 操作类型：PURCHASE/MISSED_DOSE/LOST/ADJUSTMENT |
| quantity | REAL | NOT NULL | 变更数量（正=增加，负=减少） |
| reason | TEXT | NULL | 操作原因 |
| timestamp | INTEGER | NOT NULL DEFAULT 0 | 操作时间戳（毫秒） |

#### 外键约束
```sql
FOREIGN KEY (medicationId) 
    REFERENCES medications(id) 
    ON DELETE CASCADE
```

#### 索引
```sql
-- 按药品查询日志索引
CREATE INDEX index_stock_logs_medicationId ON stock_logs(medicationId);

-- 按时间排序索引
CREATE INDEX index_stock_logs_timestamp ON stock_logs(timestamp);
```

#### 示例数据
```sql
-- 采购补货记录
INSERT INTO stock_logs (medicationId, type, quantity, reason, timestamp)
VALUES (1, 'PURCHASE', 28.0, '京东大药房采购2盒', 1717862400000);

-- 丢失消耗记录
INSERT INTO stock_logs (medicationId, type, quantity, reason, timestamp)
VALUES (1, 'LOST', -3.0, '外出遗失', 1717948800000);
```

## 3. 枚举类型

### 3.1 FrequencyType（用药频率类型）

| 枚举值 | 说明 | 日消耗量计算 |
|--------|------|-------------|
| EVERY_X_DAYS | 每X天服用 | dailyDosage / frequencyValue |
| EVERY_XTH_DAY | 每隔X天服用 | dailyDosage / (frequencyValue + 1) |

#### 计算示例
```
药品A：每2天1片 → 日消耗 = 1/2 = 0.5片/天
药品B：每隔2天1片 → 日消耗 = 1/(2+1) = 0.33片/天
```

### 3.2 StockLogType（库存操作类型）

| 枚举值 | 说明 | quantity 符号 |
|--------|------|--------------|
| PURCHASE | 采购补货 | 正数 |
| MISSED_DOSE | 漏服补回 | 正数 |
| LOST | 丢失/提前消耗 | 负数 |
| ADJUSTMENT | 手动调整 | 正数或负数 |

## 4. 核心查询

### 4.1 查询所有启用的药品
```sql
SELECT * FROM medications 
WHERE isActive = 1 
ORDER BY createdAt DESC;
```

### 4.2 查询某药品的库存日志
```sql
SELECT * FROM stock_logs 
WHERE medicationId = ? 
ORDER BY timestamp DESC;
```

### 4.3 更新库存（直接设置）
```sql
UPDATE medications 
SET currentStock = ? 
WHERE id = ?;
```

### 4.4 设置药品状态
```sql
UPDATE medications 
SET isActive = ? 
WHERE id = ?;
```

### 4.5 删除药品（级联删除日志）
```sql
DELETE FROM medications 
WHERE id = ?;
-- 关联的 stock_logs 记录会自动删除（CASCADE）
```

### 4.6 查询库存统计
```sql
-- 查询所有启用药品的总库存（按包装单位分组）
SELECT 
    packageUnit,
    COUNT(*) as drugCount,
    SUM(currentStock) as totalStock
FROM medications 
WHERE isActive = 1
GROUP BY packageUnit;
```

## 5. Room 实现

### 5.1 数据库定义
```kotlin
@Database(
    entities = [Medication::class, StockLog::class],
    version = 1,
    exportSchema = false
)
@TypeConverters(Converters::class)
abstract class AppDatabase : RoomDatabase() {
    abstract fun medicationDao(): MedicationDao
    abstract fun stockLogDao(): StockLogDao

    companion object {
        @Volatile
        private var INSTANCE: AppDatabase? = null

        fun getDatabase(context: Context): AppDatabase {
            return INSTANCE ?: synchronized(this) {
                val instance = Room.databaseBuilder(
                    context.applicationContext,
                    AppDatabase::class.java,
                    "medialert_database"
                ).build()
                INSTANCE = instance
                instance
            }
        }
    }
}
```

### 5.2 类型转换器
```kotlin
class Converters {
    private val formatter = DateTimeFormatter.ISO_LOCAL_DATE

    @TypeConverter
    fun fromLocalDate(date: LocalDate?): String? {
        return date?.format(formatter)
    }

    @TypeConverter
    fun toLocalDate(dateString: String?): LocalDate? {
        return dateString?.let { LocalDate.parse(it, formatter) }
    }
}
```

### 5.3 DAO 接口
```kotlin
@Dao
interface MedicationDao {
    @Query("SELECT * FROM medications WHERE isActive = 1 ORDER BY createdAt DESC")
    fun getAllActive(): Flow<List<Medication>>

    @Query("SELECT * FROM medications WHERE isActive = 0 ORDER BY createdAt DESC")
    fun getAllInactive(): Flow<List<Medication>>

    @Query("SELECT * FROM medications ORDER BY createdAt DESC")
    fun getAll(): Flow<List<Medication>>

    @Query("SELECT * FROM medications WHERE id = :id")
    suspend fun getById(id: Long): Medication?

    @Insert
    suspend fun insert(medication: Medication): Long

    @Update
    suspend fun update(medication: Medication)

    @Delete
    suspend fun delete(medication: Medication)

    @Query("UPDATE medications SET currentStock = :stock WHERE id = :id")
    suspend fun updateStock(id: Long, stock: Double)

    @Query("UPDATE medications SET isActive = :active WHERE id = :id")
    suspend fun setActive(id: Long, active: Boolean)
}
```

```xml

    <cloud-backup>
        <include domain="sharedpref" path="."/>
```

### 8.3 数据量预估

| 数据类型 | 预估数量 | 单条大小 | 总大小 |
|----------|----------|----------|--------|
| 药品记录 | 50条 | ~200 bytes | ~10 KB |
| 库存日志 | 500条 | ~100 bytes | ~50 KB |
| **总计** | - | - | **~60 KB** |

**结论**：数据量极小，无需分页，所有查询均可内存完成。

## 9. 安全考虑

### 9.1 数据安全
- 数据库位于应用私有目录：`/data/data/com.handy.medialert/databases/`
- 其他应用无法直接访问（需 root 权限）
- 建议用户定期导出 CSV 备份

### 9.2 敏感信息
- 药品名称属于健康信息，但非高度敏感
- 不包含身份证号、病历等敏感数据
- 符合一般健康应用的数据保护要求

### 9.3 加密建议（未来版本）
