## 2. 架构说明

### 2.1 MVVM 架构
```
UI Layer (Compose Screens/Components)
    ↕
ViewModel Layer (MedicationViewModel)
    ↕
Repository Layer (MedicationRepository)
    ↕
Data Layer (Room Database)
```

### 2.2 数据流
1. **UI** 调用 ViewModel 方法
2. **ViewModel** 调用 Repository 方法
3. **Repository** 操作 DAO
4. **DAO** 读写数据库
5. **Flow** 将数据变化反向推送到 UI

### 2.3 依赖注入
当前版本使用手动依赖注入（无 Hilt/Dagger）：
```kotlin
// Application 级别单例
class MediAlertApplication : Application() {
    val database by lazy { AppDatabase.getDatabase(this) }
    val repository by lazy { 
        MedicationRepository(
            database.medicationDao(),
            database.stockLogDao(),
            this
        )
    }
}
```

## 3. 核心模块详解

### 3.1 数据层 (data/)

#### 实体类 (entity/)
- **Medication**: 药品主实体，包含所有药品信息和库存计算逻辑
- **StockLog**: 库存变更日志，记录每次操作

#### DAO (dao/)
- **MedicationDao**: 药品的增删改查
- **StockLogDao**: 日志的查询和插入

#### 数据库 (database/)
- **AppDatabase**: Room 数据库定义
- **Converters**: LocalDate 与 String 的类型转换

### 3.2 仓库层 (repository/)

**MedicationRepository** 职责：
- 封装数据源操作
- 处理业务逻辑（如库存变更时自动记录日志）
- 提供 Flow 给 ViewModel

### 3.3 ViewModel 层 (viewmodel/)

**MedicationViewModel** 职责：
- 管理 UI 状态（StateFlow）
- 处理用户操作
- 协调 Repository 和提醒系统

### 3.4 UI 层 (ui/)

#### 页面 (screens/)
| 页面 | 职责 |
|------|------|
| MedicationListScreen | 主界面，药品列表 |
| AddMedicationScreen | 添加药品表单 |
| EditMedicationScreen | 编辑药品信息 |
| InactiveMedicationsScreen | 已停用药品列表 |
| MergedAlertScreen | 合并购药提醒 |
| SettingsScreen | 设置页面 |

#### 组件 (components/)
| 组件 | 用途 |
|------|------|
| MedicationCard | 药品卡片 |
| EmptyState | 空状态提示 |
| StatusBadge | 状态徽章 |
| StockInputDialog | 库存输入对话框 |

### 3.5 提醒系统

#### 闹钟 (alarm/)
- **AlarmScheduler**: 管理 AlarmManager
- **AlarmReceiver**: 接收广播，显示通知
- **BootReceiver**: 开机重新注册闹钟

#### 日历 (calendar/)
- **CalendarManager**: 读写系统日历
- 创建/删除购药提醒事件

## 6. 性能优化

### 6.1 数据库优化

#### 添加索引
```kotlin
// 在 MedicationDao.kt 中
@Query("SELECT * FROM medications WHERE isActive = 1 ORDER BY createdAt DESC")
@Transaction  // 复杂查询使用事务
fun getAllActive(): Flow<List<Medication>>
```

#### 分页加载
```kotlin
// 未来版本可添加分页
@Query("SELECT * FROM medications WHERE isActive = 1 ORDER BY createdAt DESC LIMIT :limit OFFSET :offset")
suspend fun getActivePaged(limit: Int, offset: Int): List<Medication>
```

### 6.2 Compose 优化

#### 避免不必要的重组
```kotlin
// 使用 remember 缓存计算结果
val sortedMedications = remember(medications) {
    medications.sortedBy { it.daysUntilDepletion() }
}

// 使用 derivedStateOf 减少重组频率
val hasUrgent by remember {
    derivedStateOf { medications.any { it.daysUntilDepletion() <= 1 } }
}
```

#### 延迟加载
```kotlin
// 图片/复杂组件延迟加载
LazyColumn {
    items(medications, key = { it.id }) { medication ->
        MedicationCard(
            medication = medication,
            // ...
        )
    }
}
```

### 6.3 提醒系统优化

#### 批量操作
```kotlin
// 批量注册闹钟，减少系统调用
fun rescheduleAllAlarms(medications: List<Medication>) {
    medications.filter { it.isActive }.forEach { medication ->
        cancelAlarm(medication.id)
        scheduleAlarm(medication)
    }
}
```

## 8. 发布流程

### 8.1 版本号规范
遵循 [语义化版本](https://semver.org/lang/zh-CN/)：
- **MAJOR**: 不兼容的 API 修改
- **MINOR**: 向下兼容的功能新增
- **PATCH**: 向下兼容的问题修正

### 8.2 发布检查清单

- [ ] 更新 `versionCode` 和 `versionName`
- [ ] 运行所有测试
- [ ] 检查 ProGuard 规则
- [ ] 验证权限声明
- [ ] 测试深色模式
- [ ] 测试不同屏幕尺寸
- [ ] 生成签名 APK

### 8.3 生成发布 APK

```bash
# 创建签名密钥（仅需一次）
keytool -genkey -v -keystore medialert.keystore -alias medialert -keyalg RSA -keysize 2048 -validity 10000

# 配置签名（app/build.gradle.kts）
android {
    signingConfigs {
        create("release") {
            storeFile = file("medialert.keystore")
            storePassword = "your_password"
            keyAlias = "medialert"
            keyPassword = "your_password"
        }
    }
    buildTypes {
        release {
            signingConfig = signingConfigs.getByName("release")
            isMinifyEnabled = true
            proguardFiles(...)
        }
    }
}

# 构建发布版
./gradlew assembleRelease
```
