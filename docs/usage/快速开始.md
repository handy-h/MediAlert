# 药箱库存管家 - 快速开始指南

## 1. 环境准备

### 1.1 系统要求
- **操作系统**: Windows 10/11, macOS, 或 Linux
- **JDK**: JDK 17 或更高版本
- **Android Studio**: Hedgehog (2023.1.1) 或更高版本
- **Android SDK**: API 26 (Android 8.0) 到 API 34 (Android 14)

### 1.2 安装 Android Studio
1. 访问 [Android Studio 官网](https://developer.android.com/studio)
2. 下载并安装最新版 Android Studio
3. 启动 Android Studio，完成首次配置向导

### 1.3 配置 JDK
确保系统环境变量 `JAVA_HOME` 指向 JDK 17：

```bash
# Windows PowerShell
$env:JAVA_HOME
# 应输出类似: C:\Program Files\Java\jdk-17

# 验证 Java 版本
java -version
# 应输出: openjdk version "17" 或更高
```

## 2. 项目导入

### 2.1 打开项目
1. 启动 Android Studio
2. 选择 **Open**（打开现有项目）
3. 导航到 `D:\Builds\MediAlert` 目录
4. 点击 **OK** 等待 Gradle 同步完成

### 2.2 Gradle 同步
首次打开时，Android Studio 会自动：
- 下载 Gradle Wrapper
- 下载项目依赖
- 编译项目

如果同步失败，请检查：
- 网络连接（需要访问 Google Maven 仓库）
- `gradle.properties` 中的代理配置（如需要）

### 2.3 解决常见同步问题

#### 问题：Gradle 下载超时
**解决**：在 `gradle.properties` 中添加：
```properties
org.gradle.jvmargs=-Xmx4096m -Dfile.encoding=UTF-8
org.gradle.daemon=true
org.gradle.parallel=true
org.gradle.configureondemand=true
```

#### 问题：依赖下载失败
**解决**：检查 `settings.gradle.kts` 中的仓库配置：
```kotlin
dependencyResolutionManagement {
    repositories {
        google()
        mavenCentral()
    }
}
```

## 3. 构建与运行

### 3.1 构建调试版本

```bash
# 在 Android Studio 终端或系统终端中执行
./gradlew assembleDebug
```

构建成功后，APK 位于：
```
app/build/outputs/apk/debug/app-debug.apk
```

### 3.2 连接真机或启动模拟器

#### 真机调试
1. 开启手机 **开发者选项**：
   - 设置 → 关于手机 → 连续点击"版本号"7次
2. 开启 **USB 调试**：
   - 设置 → 系统 → 开发者选项 → USB 调试
3. 用 USB 线连接电脑，允许调试授权

#### 模拟器调试
1. 在 Android Studio 中打开 **Device Manager**
2. 点击 **Create Device**
3. 选择手机型号（推荐 Pixel 7）
4. 下载系统镜像（推荐 API 34）
5. 启动模拟器

### 3.3 运行应用

#### 方式一：Android Studio 直接运行
1. 点击工具栏的 **Run** 按钮（绿色三角形）
2. 或按 `Shift + F10`
3. 选择目标设备（真机或模拟器）

#### 方式二：命令行安装
```bash
# 构建并安装
./gradlew installDebug

# 或先构建再安装
./gradlew assembleDebug
adb install app/build/outputs/apk/debug/app-debug.apk
```

### 3.4 查看日志
```bash
# 过滤应用日志
adb logcat -s "MediAlert" "AndroidRuntime" "System.err"

# 或查看所有日志
adb logcat
```

## 4. 首次使用

### 4.1 权限授予
首次启动时，应用会请求以下权限：

| 权限 | 用途 | 是否必须 |
|------|------|----------|
| 日历读写 | 创建购药提醒事件 | 是 |
| 精确闹钟 | 设置库存预警闹钟 | 是 |
| 通知 | 显示提醒通知 | 是 |

**操作**：点击"允许"授予所有权限。

### 4.2 添加第一盒药品

1. 点击右下角 **+** 按钮
2. 填写药品信息：
   - **通用名**：厄贝沙坦（必填）
   - **商品名**：安博维（可选）
   - **规格**：80/12.5（可选）
   - **包装**：盒
   - **剂型**：片
   - **每盒含量**：14
   - **当前库存**：2盒 + 6片
   - **用药频率**：每1天
   - **每次剂量**：1片
3. 点击 **保存药品**

### 4.3 查看库存状态
- 主界面显示所有药品卡片
- 卡片右上角显示状态徽章：
  - 🔴 红色：≤1天耗尽
  - 🟡 黄色：≤4天耗尽
  - 🟢 绿色：>4天耗尽

### 4.4 设置提醒
1. 点击右上角 **设置** 图标
2. 点击 **日历账户**
3. 授予日历权限
4. 选择要绑定的日历账户
5. 应用会自动为所有药品注册提醒

## 5. 日常使用

### 5.1 补货
1. 在药品卡片上点击 **补货**
2. 输入新增数量（如：3盒 + 0片）
3. 可选填写原因（如"京东采购"）
4. 点击 **确认**
5. 库存自动更新，提醒重新计算

### 5.2 消耗库存
1. 在药品卡片上点击 **消耗**
2. 输入消耗数量（如：0盒 + 3片）
3. 可选填写原因（如"丢失"）
4. 点击 **确认**

### 5.3 停用药品
1. 在药品卡片上点击 **停用**
2. 药品移至"已停用"列表
3. 相关提醒自动取消

### 5.4 重新启用
1. 点击主界面右上角 **归档** 图标
2. 找到要启用的药品
3. 点击 **▶ 播放** 图标
4. 药品恢复计算，提醒重新注册

### 5.5 查看合并提醒
1. 点击主界面右上角 **通知** 图标
2. 查看按日期合并的购药提醒
3. 每个卡片显示：
   - 提醒日期（含工作日/周末标识）
   - 该日需要购买的所有药品列表

### 5.6 导出数据
1. 进入 **设置**
2. 点击 **导出数据**
3. CSV 文件保存到 `/Download/药箱库存_时间戳.csv`
4. 可用 Excel 或记事本打开

## 6. 常见问题

### Q1: 提醒没有触发？
**检查**：
1. 是否授予了日历和闹钟权限？
2. 手机是否开启了省电模式？（可能限制后台）
3. 日历事件是否成功创建？（检查系统日历应用）

### Q2: 库存计算不准确？
**检查**：
1. 用药频率设置是否正确？
2. 开始日期是否设置？（不设置则按今天计算）
3. 每包装含量是否填写正确？

### Q3: 如何修改包装信息？
**答**：目前不支持直接修改包装信息。需要：
1. 删除旧药品
2. 重新添加（库存会自动迁移）

### Q4: 换手机如何迁移数据？
**答**：v1.0 暂不支持云同步。可：
1. 使用 **导出数据** 功能导出 CSV
2. 新手机上手动重新录入
3. 未来版本将支持云备份

### Q5: 支持哪些剂型？
**答**：当前支持：
- 片
- 粒
- ml（液体）
- g（粉末/膏体）

如需其他剂型，请在设置中反馈。

## 7. 卸载

### 7.1 卸载应用
```bash
adb uninstall com.handy.medialert
```

### 7.2 清理残留数据
卸载后，以下数据将被清除：
- 本地数据库（所有药品和日志）
- 应用设置
- 已注册的日历事件（需手动清理）

**建议**：卸载前请先 **导出数据** 备份。
